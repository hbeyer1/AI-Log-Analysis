You are analyzing a record of someone's conversation with an AI assistant. Your job is to identify each distinct objective the user had in this session and describe how each was pursued.

<input>
Session ID: {session_id}

Transcript (each turn is labelled with its 0-based index and role):

{transcript}
</input>

<existing_objectives>
Objectives already extracted from earlier chunks of this session. Continue them rather than re-extracting. If the most recent one is marked "in progress", extend it with any new turns in this chunk instead of creating a duplicate.

{existing_objectives}
</existing_objectives>

<instructions>
1. Read the entire transcript. Identify each DISTINCT objective the user brought to the assistant. An objective is a coherent goal: fix a specific bug, draft a specific email, understand a specific concept. A follow-up clarification on the same goal is NOT a new objective. A session may contain one or several.

2. For each objective, write a RESOLUTION SUMMARY describing how the user pursued it, covering all four dimensions in a single paragraph:
   (a) Initial framing — How did the user present the task? (Concrete ask? Open exploration? With or without context/examples?)
   (b) Interaction pattern — Single exchange or iterative? If iterative, what drove the iterations? (corrections, refinements, follow-up questions, new requirements)
   (c) User effort — How many user turns were spent on this objective.
   (d) Outcome — One of: resolved, partially resolved, abandoned, unclear.

3. Stay faithful to the user's framing. Do not paraphrase away specifics, and do not impose categories — this step is supposed to be faithful to the data, not pre-labelled. Use the user's vocabulary where possible.

4. source_quote must be a VERBATIM substring of a user message — copy it exactly from the transcript. Choose a short, representative phrase (roughly 5-20 words). The quote will be substring-checked; paraphrased quotes are rejected.

5. turn_indices refer to the `Turn N` numbers shown in the header of each message. Include ALL user-turn indices that belong to this objective. Do NOT include assistant turns.

6. Do not extract objectives from assistant turns. The assistant is not the subject of this analysis; the user is.
</instructions>

<output_format>
Return a JSON array. One object per objective, in the order they appear in the transcript.
Schema per object:

{
  "objective": "<concise description of the goal, <=200 chars>",
  "resolution_summary": "<one paragraph covering (a) initial framing, (b) interaction pattern, (c) user effort in turns, (d) outcome. Approx 40-100 words.>",
  "source_quote": "<verbatim user text>",
  "timestamp": "<ISO-8601 timestamp of the initiating user message>",
  "timestamp_range": ["<first turn ISO-8601>", "<last turn ISO-8601>"],
  "turn_indices": [<0-based indices of user turns belonging to this objective>]
}

If the segment contains no meaningful objective, return an empty array: []
Return ONLY the JSON array. No prose, no markdown fences.
</output_format>
